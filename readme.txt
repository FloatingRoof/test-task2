1. Некоторые цвета (которые берутся с API) отличаются от макета.
2. API сделана не очень логично, цвета статусов хранятся прямо вместе с заявками, а цвета приоритета хранятся в другом запросе?
3. Я не стал устанавливать шрифты, которые были установлены в psd, не хватило времени, но зато остальные задачи я выполнил практически все.
4. В описаниях task присутсвуют HTML-теги, что является небезопасным способом, реализовать показ тега из текста небезопасно, поэтому нужно использовать другой подход в этой задаче, и не хранить такие данные в БД.
5. Непонятная реализация комментариев. В макете дан комментарий под кнопкой сохранить, но в запросах нигде нет информации, как получать комментарии для определенной задачи, поэтому этот блок я не сделал.
6. В макете есть 3 поля: Заявитель, Создатель и исполнитель. В API всего два поля, это автор и исполнитель, поэтому лишнее поле убрал.
7. Чтобы сохранить данные (статус и исполнитель) нужно нажать кнопку сохранить (при этом должен идти и запрос на добавление комментария, но это не предусмотренно сервером).
8. В задании не сказано, какой должен быть статус при создании задачи, поэтому я его не настраивал.
9. Для state менеджмента я использовал Redux.
10. Доп. библиотеки которые я использовал: axios, classnames, dateformat, formik, react-redux, react-router-dom, redux, redux-thunk, yup.
